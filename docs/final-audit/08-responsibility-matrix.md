# 08 — Матрица ответственности

«Никита» = разработчик (технические задачи, которые делаются через Claude Code/кодом). «Владелец» = вы (то, что нельзя/нецелесообразно автоматизировать).

| Процесс | Система (авто) | Никита (разработчик) | Владелец ONYX | Как часто |
|---|---|---|---|---|
| Поддержание кода | — | правит `api/index.py` | — | по мере |
| Исправление ошибок | лог ошибок в `/admin` | диагностика+фикс | сообщить о симптоме | по мере |
| Деплой | авто при пуше (Vercel) | коммит+пуш (или инструкция) | нажать Push в GitHub Desktop / дать доступ | по релизу |
| Резервные копии | Sheets = журнал; KV на Upstash | настроить экспорт/бэкап | подтвердить план хранения | еженедельно |
| Контроль интеграций | — | сверить PR-CY/AI/Sheets/Cron | дать ключи | при настройке |
| Обновление тарифов | — | правит `TARIFFS` + пуш | утвердить цены/состав | по изменению |

## Что должен доделать Никита (техническое — НЕ перекладывать на владельца)
1. Подключить Core-слой (раздельные ID, application-сущность, журнал) к живым хендлерам + миграция.
2. Сохранение прогресса анкеты при уходе в тарифы и после `/start` (экран «продолжить»).
3. Версии анкет/заявок в живом флоу (не перезаписывать `onyx:quest`).
4. Мультизаявочный кабинет + выбор заявки в «Статусе».
5. Единый источник тарифов/опций (конфиг), чтобы синхронизировать с сайтом.
6. Полная сводка заявки перед подтверждением (тариф+опции+анкета+реквизиты).
7. Авто-статус/событие на выдаче чек-листа; уведомление админу «начал анкету».
8. Адаптер внутреннего AI-агента аудита + очередь + fallback.
9. Интеграция сайт→бот (deep-link кнопки, форма→лид, wa-токен) — бот-сторона готова, нужна сайт-сторона.
10. Живой смоук-тест после пуша; подтвердить компиляцию; починить, если что-то падает.
11. Экран сравнения тарифов; поле «в какой тариф входит» у опции.
12. Написать/держать актуальными инструкции и тесты.

## Что должен сделать владелец (только неавтоматизируемое)
1. Выдать доступы/ключи (см. `09`): `ADMIN_TELEGRAM_IDS`, PR-CY, AI, Google service account, доступ к коду сайта, WhatsApp, данные «Мой налог».
2. Подтвердить цены/состав тарифов и опций.
3. Юридические данные/согласия (политика ПДн, оферта).
4. Ручная связь с клиентом в WhatsApp и проведение консультаций.
5. Утверждение бизнес-решений и утверждение сайта клиентом.
6. Выставление чека через «Мой налог» и подтверждение факта оплаты.
7. Ручные производственные статусы, предоставление материалов, подтверждение публикации.
8. Нажимать Commit/Push в GitHub Desktop (пока не настроен CI-доступ) и давать разрешение на прод-деплой.
