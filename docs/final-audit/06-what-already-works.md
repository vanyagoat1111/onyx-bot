# 06 — Что в ONYX уже работает

Здесь только функции, у которых цепочка «в коде» связна и логика проверена (code/iso). **Все они требуют одного живого прогона после пуша** (Л2) — до этого «работает» = «должно работать по коду». Частичные и неподключённые функции сюда НЕ включены.

### 1. Приём клиента и меню
- **Что:** `/start` регистрирует клиента, показывает приветствие и меню (аудит, тарифы/услуги, мой заказ, кабинет, партнёр, поддержка).
- **Куда нажать:** открыть бота, `/start`.
- **Результат:** профиль создаётся (`onyx:user`), доступно меню.
- **Где данные:** KV + лист Clients.

### 2. Анкета 27 вопросов
- **Что:** пошаговая анкета одним редактируемым сообщением, прогресс-бар, галочки, «Пропустить»/«Отменить»/«Назад», резюме + подтверждение.
- **Куда:** «🌐 Разработать сайт» / «Заполнить анкету».
- **Результат:** questionnaire_status=filled, данные в KV + лист Questionnaire, уведомление админу.
- **Ограничение:** прогресс теряется, если уйти в тарифы или сделать `/start` (см. backlog).

### 3. Тарифы (3 пакета) и опции
- **Что:** «🛒 Тарифы и услуги» → «Тарифы» (Старт 5 990 / Сайт+заявки 8 900 / Система от 19 900, без сопровождения) и «Доп. опции». Выбор тарифа сохраняется, опции в корзину, сумма считается.
- **Результат:** chosen_tariff в профиле, сумма = тариф + опции.

### 4. Оформление заявки → консультация (без предоплаты)
- **Что:** после анкеты → реквизиты (физ/юр) → заявка в статус «📞 Ожидает консультации». Клиенту объясняется: оплата через «Мой налог» только после утверждения сайта.
- **Результат:** заказ (`onyx:order`) создан, задача «провести консультацию», уведомление админу.

### 5. Статусы проекта («Мой заказ»)
- **Что:** клиент видит понятный статус (консультация → разработка → показ → правки → утверждён → счёт → оплачен → опубликован → завершён), описание и срок.
- **Куда:** «📦 Мой заказ».

### 6. Бесплатный аудит + реестр
- **Что:** ввод URL → нормализация домена → поиск в реестре: свежий отдаётся повторно, устаревший предлагает обновить, нового — запускает PR-CY (нужен ключ). AI-резюме или безопасный шаблон. Кнопки: получить предложение / исправления / новый сайт.
- **Ограничение:** реальный PR-CY требует боевого ключа (🔌).

### 7. Отзывы 1–10
- **Что:** оценка 1–10 → комментарий → видеокружок → разрешение → «Спасибо за ваш отзыв, это помогает ONYX расти». Низкая оценка → задача разобрать.

### 8. Партнёрская программа
- **Что:** «🤝 Стать партнёром» → анкета → партнёрский код. Реферальные ссылки `ref<id>`.

### 9. Поддержка + FAQ + тикеты
- **Что:** «🆘 Поддержка» → написать в поддержку / создать обращение (категория+текст) / мои обращения / FAQ (16 вопросов, включая «Мой налог»). Админ отвечает `/reply <id> <текст>`.

### 10. Контент-подписки и рассылки
- **Что:** подписка на темы, админ-рассылка `/post` по темам с предпросмотром, `/broadcast` всем.

### 11. Админ-панель `/admin`
- **Что:** статистика, воронка, клиенты по тегам, клиенты для дожима, заказы, задачи, производство, домены, партнёры, ошибки. Смена статусов заказа кнопками. `/export` — сводка.

### 12. Операционная автоматизация (фон)
- **Что:** лиды+температура, авто-теги, автодожимы (Cron: брошенная анкета/корзина/аудит/тишина), задачи команды с автосозданием, производственный конвейер при оплате, доменные записи, лог ошибок.

### 13. Deep-link распознавание (бот-сторона)
- **Что:** бот понимает `t_`, `o_`, `build`, `audit`, `checklist`, `lm_`, `wa_`, `ref` и ведёт по нужному сценарию. **Но:** сайт пока эти ссылки не формирует — до интеграции сайта работает только при ручном формировании ссылки.

> Что сюда НЕ вошло (частично/не подключено): раздельные ID и журнал (Core), версии анкет/заявок, вход с сайта/WhatsApp сквозняком, сохранение прогресса анкеты, мультизаявочный кабинет, сравнение тарифов, авто-статус чек-листа.
