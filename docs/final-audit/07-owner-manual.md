# 07 — Как пользоваться ботом и автоматизацией ONYX (для владельца)

Простым языком, без кода. Всё, что помечено 🔧, требует, чтобы вы один раз настроили доступы (см. `09-missing-owner-inputs.md`).

## 1. Как запустить систему
- **Бот:** Telegram-бот ONYX (токен в переменной `BOT_TOKEN` на Vercel). Работает как «webhook» на Vercel — отдельный сервер запускать не нужно.
- **Сервер:** Vercel, проект `onyx-bot-4xn3.vercel.app`. Деплой — автоматически при пуше в GitHub (ветка main).
- **База:** Upstash Redis (KV) + Google-таблица (панель).
- **Что должно быть включено:** переменные окружения `BOT_TOKEN`, `KV_REST_API_URL/TOKEN`, `SHEETS_WEBHOOK_URL`, `ADMIN_TELEGRAM_IDS`, `CHECKLIST_URL`; включён Vercel Cron.
- **Проверить, что работает:** напишите боту `/start` — если ответил и показал меню, бот жив. Команда `/admin` (под вашим ID) откроет панель.

## 2. Как проверить нового клиента
- Клиент появляется, как только напишет боту `/start`.
- **Где смотреть:** Google-таблица, лист **Clients** (имя, телефон, ниша, статус анкеты, источник). В боте — `/admin` → «Статистика».
- **Источник перехода:** колонка source (откуда пришёл: тариф/аудит/чек-лист/whatsapp/реклама).
- **Действия клиента:** лист **Events** (если включён `EVENTS_TO_SHEET=1`) и `/admin` → «Воронка».
- **Чек-лист/аудит/анкета/тариф/опции:** листы Audits, Questionnaire, Orders.

## 3. Заявка с сайта (когда будет интеграция сайта)
- Сейчас **вход с сайта не подключён** (нужен код на сайте — см. backlog). Когда подключим: заявка попадёт в лист **Leads/ClientsCore** со статусом «Нужно связаться в WhatsApp», вам придёт уведомление в Telegram.
- **Что делать:** написать клиенту в WhatsApp, отправить персональную ссылку в бота (её сгенерит система). Клиент перейдёт — его Telegram свяжется с заявкой автоматически.

## 4. Как работать с анкетой
- **Где ответы:** лист **Questionnaire** (30 полей) + уведомление админу при заполнении.
- **Завершена ли:** статус `filled` в Clients/Questionnaire.
- **Тариф/опции/сумма:** лист **Orders** (после оформления заявки).

## 5. Личный кабинет клиента
- Создаётся автоматически при первом контакте. Найти клиента — по Telegram ID/имени в листе Clients.
- **Заявки клиента:** лист Orders (по telegram_id).
- **Сменить статус:** в боте `/admin` → «Заказы» → выбрать → кнопки статуса; или команда `/set_order_status <№>`.
- **Отправить обновление:** смена статуса сама уведомляет клиента.

## 6. Аудиты
- **Готовый аудит:** лист **Audits** / в боте реестр по домену (повторный запрос того же сайта отдаёт готовый).
- **Новый:** клиент вводит URL — система сама запускает (нужен ключ PR-CY 🔧). Результат сохраняется и отправляется клиенту.
- **Ошибка:** если PR-CY недоступен — статус «нужен ручной разбор» + уведомление вам; сделайте аудит вручную и напишите клиенту.

## 7. Консультация
- Когда клиент оформил заявку — приходит уведомление «🔥 Новая заявка — нужна консультация» и создаётся задача (`/admin` → «Задачи»).
- **Что делать:** связаться с клиентом, обсудить проект. После — смените статус заказа на «Сайт в разработке».

## 8. Разработка сайта
- Статусы меняете вы вручную в `/admin` → «Заказы»: Изучаем анкету → В разработке → Готов к показу → Показан → Правки → **Утверждён**.
- Каждая смена статуса автоматически уведомляет клиента.
- Автоматически меняются только лид-статусы/теги; производственные статусы — вручную.

## 9. Счёт через «Мой налог»
- Как только вы ставите статус **«✅ Сайт утверждён»**, система создаёт задачу «Выставить счёт через Мой налог» и уведомляет вас.
- **Что делать:** в приложении «Мой налог» сформируйте чек на сумму заявки (видна в Orders), отправьте клиенту. В боте поставьте статус «🧾 Счёт выставлен».
- **Подтвердить оплату:** команда `/paid <№заказа>` — клиент получит уведомление, запустится производство.

## 10. Завершить заказ
- После публикации поставьте статус «🚀 Опубликован», затем «🎉 Завершён» — клиенту уйдёт финальное сообщение и запрос отзыва. История сохраняется в Orders (старые заявки не удаляются).

## 11. Менять тарифы и опции
- Сейчас тарифы/опции и цены **зашиты в коде** (`TARIFFS` и `SERVICES` в `api/index.py`). Изменение цены/описания = правка кода + пуш. **Единого источника с сайтом пока нет** — при изменении правьте и на сайте, и в боте (это в backlog как P1).

## 12. Что делать при ошибках
- **Бот не отвечает:** проверьте деплой на Vercel и `BOT_TOKEN`; посмотрите Runtime Logs Vercel.
- **Заявка не записалась:** проверьте `SHEETS_WEBHOOK_URL` и KV; в боте `/admin` → «Ошибки».
- **Статус не изменился:** проверьте, что вы админ (`ADMIN_TELEGRAM_IDS`).
- **Клиент не найден:** ищите по telegram_id в Clients.
- **Аудит не создался:** проверьте `PRCY_API_KEY`.
- **Уведомление не пришло:** проверьте `ADMIN_TELEGRAM_IDS` (ваш ID).
- **Дубли:** возможны, пока не подключён Core-дедуп (backlog).
- **Сервер остановился:** Vercel сам поднимает функции; при постоянных ошибках — Runtime Logs.

> Точные команды админа: `/admin`, `/admin_help`, `/orders`, `/order N`, `/set_order_status N`, `/paid N`, `/reply <id> <текст>`, `/post`, `/broadcast`, `/partners`, `/export`, `/id`.
