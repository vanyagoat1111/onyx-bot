## Л1. Не все 18 промптов доступны как отдельные тексты
Через рабочую сессию фактически прошли и доступны дословно:
- **ТЗ по боту** — файл «оникс промпт по боту.txt» (анкета 27, тарифы/корзина, оплата, кабинет, «Мой заказ», статусы, подписка, аудит, отзывы, партнёрка, контент-рассылки, Google Sheets, уведомления).
- **ТЗ операционного центра** — файл «оникс промптик.txt» (админ-панель, лиды, дожимы, производство, дизайн-модуль, домены, задачи, тикеты, FAQ, предложения после аудита, теги, Events, защита от дублей, экспорт).
- **Правки бизнес-модели** — отзыв 1–10; отказ от Prodamus → счета; отключение сопровождения; кнопка «Личный кабинет» после анкеты.
- **Архитектурный промпт** → создан `onyx_docs/ONYX_BOT_ARCHITECTURE_2026-07-20.md`.
- **Промпт реализации** (15 этапов).
- **Этот приёмочный промпт** (18-й).

**Отсутствуют как отдельные пронумерованные тексты:** ранние «15 промптов» по сайту/квизам/шаблонам/живым демо в исходной формулировке. Реестр требований построен по доступным материалам + живому сайту onyx-web.ru (изучен) + коду. Если у вас есть эти 15 промптов — пришлите, дополню реестр (сейчас помечено `SRC=session` / `SRC=site` / `SRC=missing?`).

## Л2. Настоящие сквозные тесты не выполнялись
- Нет доступа к **запущенному боту** в Telegram (нет возможности послать `/start` и пройти сценарий).
- Примонтированная копия репозитория в песочнице **зависла на снимке начала сессии** — `python -m py_compile api/index.py` по актуальному файлу выполнить нельзя. **Не подтверждено даже, что текущий файл компилируется.**
- Поэтому во всех «сквозных тестах» (`03-...`) колонка «Фактический результат» = **трассировка по коду / изолированный модульный тест**, а не живой прогон. Живые прогоны — за владельцем/разработчиком после пуша.

## Л3. Состояние рабочего дерева на момент аудита
- Весь объём этой сессии (анкета 27, операционный центр, отказ от Prodamus/сопровождения, deep-link, тарифы-пакеты, гейт оплаты, реестр аудитов, а также **начатый, но не подключённый Core-слой данных**) находится в рабочем файле `api/index.py`.
- **Не подтверждено, что всё это закоммичено, запушено и задеплоено.** Задеплоенный бот может быть старее рабочего файла. До пуша+живого прогона считать реализованное «в коде», а не «в проде».
- Изолированные тесты логики (анкета, лиды/температура, задачи, дожимы, deep-link, тарифы+сумма, гейт оплаты, реестр аудитов) — **проходили**. Это не заменяет живой прогон.

## Что это значит для оценок готовности
Проценты в `11-readiness-summary.md` даны для **кода**. Готовность «к реальным клиентам» отдельно занижена, потому что не пройден ни один живой сквозной сценарий и не подтверждена компиляция/деплой.
